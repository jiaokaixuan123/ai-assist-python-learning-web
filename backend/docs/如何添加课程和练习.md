## 课程内容编写规范

### Markdown 格式

- 使用 `##` 作为主标题
- 使用 `###` 作为小节标题
- 代码块用三个反引号包裹，指定语言 `python`
- 表格使用 Markdown 表格语法

### 示例模板

```markdown
## 主题名称

简短介绍（1-2 句话）。

### 基本概念

解释核心概念，配合代码示例：

\`\`\`python
# 示例代码
x = 10
print(x)
\`\`\`

### 常见用法

列举 2-3 个常见场景。

### 注意事项

- 注意点 1
- 注意点 2

### 练习

给出一个小练习，让学生动手实践。
```

## 测试用例编写规范

### 格式

```python
{
    "input": "输入数据（可选）",
    "expected_output": "预期输出（必填）"
}
```

### 注意事项

1. `expected_output` 必须与 `print()` 输出完全匹配（包括空格、换行）
2. 多行输出用 `\n` 分隔
3. 如果不需要输入，`input` 留空字符串

### 示例

```python
# 简单函数测试
{"input": "", "expected_output": "8"}

# 多行输出
{"input": "", "expected_output": "Hello\nWorld"}

# 列表输出
{"input": "", "expected_output": "[1, 2, 3, 4, 5]"}
```

## 知识库集成（RAG）

### 1. 安装依赖

```bash
pip install sentence-transformers chromadb openai
```

### 2. 构建向量数据库

```bash
cd backend
python build_knowledge_base.py
```

这会将所有课程内容向量化并存入 `./chroma_db`。

### 3. 使用知识库 API

**检索相关知识：**

```bash
POST /api/knowledge/search
{
  "query": "如何使用列表推导式",
  "top_k": 3
}
```

**基于知识库问答：**

```bash
POST /api/knowledge/ask
{
  "question": "Python 中如何反转字符串？",
  "model_api_key": "sk-..."
}
```

### 4. 更新知识库

每次添加新课程后，重新运行：

```bash
python build_knowledge_base.py
```

## 常见问题

### Q: 如何修改已有课程？

A: 使用 MongoDB 客户端或编写更新脚本：

```python
await db.courses.update_one(
    {"_id": ObjectId(course_id)},
    {"$set": {"description": "新描述"}}
)
```

### Q: 如何删除课程/练习？

A:

```python
await db.courses.delete_one({"_id": ObjectId(course_id)})
await db.exercises.delete_one({"_id": ObjectId(exercise_id)})
```

### Q: 测试用例不通过怎么办？

A: 检查：
1. 输出格式是否完全匹配（包括空格）
2. 是否有多余的 `print()` 语句
3. 数据类型是否正确（字符串 vs 数字）

### Q: 知识库检索不准确？

A: 尝试：
1. 增加 `top_k` 参数
2. 优化课程内容的结构和关键词
3. 使用更好的向量模型（如 `text-embedding-ada-002`）
